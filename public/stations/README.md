# Stations (offline app)

Each `station-N/` folder holds the editable content for one station:

- `<station name>.txt` — an empty "marker" file whose **filename** is the
  station's display name (e.g. `Hurra! Emancipation.txt`). Doubles as a
  human-friendly label when browsing the folder in Finder. The contents are
  ignored; rename the file to rename the station.
- `description.txt` — **short** description (1–3 sentences). Shown in italic
  *above the clue* on the locked station page, before the QR is scanned.
- `description-long.txt` — **long** description (a few paragraphs). Shown
  *below the photo carousel and the audio player* once the station is
  unlocked. Blank lines split it into paragraphs.
- `clue.txt` — hint to where the QR code is hidden
- `images/` — photos shown in the unlocked-state carousel, named
  `image-1.jpg`, `image-2.jpg`, … (see `images/README.md`)

`manifest.json` in this folder is auto-generated by
`scripts/generate-stations-manifest.js` (runs on `npm start` / `npm run build`)
and maps each station folder → the marker filename.

Structural data (id, category, QR unlock token) lives in the `STATIONS` array
inside `src/offline-app.jsx` — edit there if you swap a station for a
different place. The folder names stay stable so you don't have to rename
folders each time.

Audio files live in `public/audio/` and must start with the station number,
e.g. `01-anything.mp3` → station 1. The exact filename is resolved at runtime
via `public/audio/manifest.json` (also auto-generated).
